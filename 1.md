## Что делает этот очиститель

| Действие | Пример | Статистика |
|---|---|---|
| Удаляет пустые | `""` | `empty_removed` |
| Удаляет URL | `"См. https://example.com"` | `garbage_pattern_removed` |
| Удаляет ISBN/УДК/ББК | `"ISBN 978-5-17-123456-7"` | `garbage_pattern_removed` |
| Удаляет email | `"info@example.com"` | `garbage_pattern_removed` |
| Удаляет предметные указатели | `"James, 216\nChapel, 382"` | `index_removed` |
| Удаляет строки без букв (>50% цифр) | `"123 456 789"` | `high_digit_removed` |
| Удаляет точные дубли подряд | `"Привет."` → `"Привет."` | `adjacent_dupes_removed` |
| Убирает лишние пробелы | `"текст   текст"` → `"текст текст"` | — |

## Что НЕ делает (сознательно)

| Не трогаем | Причина |
|---|---|
| Кавычки `«» "" „“` | Сохраняем стилистику |
| Тире `— – -` | Сохраняем как есть |
| Маркеры списков `1. 1) -` | Сохраняем структуру |
| Короткие предложения (`<20`) | Диалоги важны |
| Греческие символы | Нужны для науки |
| Математические символы | Нужны для науки |

## Сборка дата сета для обучения и валидации, часть от каждой книги попадает в валидационный сет
### Стратегия
Для каждой книги:
- 98% предложений → train чанки
- 2% случайных предложений → val чанки

Это гарантирует, что val покрывает все жанры, всех авторов, все стили.

1. Каждая книга даёт и train, и val чанки
2. 2% случайных предложений помечаются как val
3. Чанки строятся отдельно для train и val (чтобы не смешивать предложения в одном чанке)
4. На выходе 6 файлов:
    - phase1_128_train.txt, phase1_128_val.txt
    - phase2_256_train.txt, phase2_256_val.txt
    - phase3_512_train.txt, phase3_512_val.txt
